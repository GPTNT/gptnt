# Internals

There are reasons why things are the way they are. Through all of these, we try to explain the underlying reasoning behind the design decisions and implementation choices. This is not a complete list of all the internals, but it is a good starting point to understand how things work, and how to avoid the footguns that we have encountered along the way.

If you are going to extend the benchmark or add new features, this section is for you. It is also a good place to start if you are trying to understand how the benchmark works, and how to debug it. However, it will focus on how things _are_—making light reference to previous iterations and learnings.
